# NZXT-ESC

### Расширенный редактор макетов и настройки экрана для NZXT Kraken AIO

Создавайте полностью редактируемые LCD-макеты NZXT Kraken с перетаскиваемыми сенсорными оверлеями, собственными шрифтами, изображениями, GIF, MP4-видео, часами, графиками, данными Now Playing и визуализациями, реагирующими на звук, с выводом в реальном времени через **NZXT CAM Web Integration**.

## Настройка LCD NZXT Kraken без фиксированных макетов

NZXT-ESC превращает дисплей NZXT Kraken в свободное полотно. Создайте собственный LCD-экран, разместив каждый датчик, график, часы, изображение или медиаэлемент точно там, где нужно. Изменяйте размер, вращайте, переставляйте, переименовывайте, блокируйте и оформляйте элементы, наблюдая за обновлением результата в реальном времени через NZXT CAM.

Основной редактор **не требует учетной записи** и **не требует отдельной установки для конечного пользователя**. Пресеты и локальные медиа остаются в хранилище браузера. Дополнительные музыкальные оверлеи используют локальное Windows-приложение [NowPlaying.WebSocket](https://github.com/mrgogo7/NowPlaying.WebSocket).

## Быстрый старт

### Открыть напрямую в NZXT CAM

[![Open NZXT-ESC in NZXT CAM](https://img.shields.io/badge/Open%20NZXT--ESC%20in-NZXT%20CAM-8b5cf6?style=for-the-badge)](https://cam-redirect.nzxt.com/action/load-web-integration?url=https://nzxt-esc.pages.dev/)

1. Нажмите **Open NZXT-ESC in NZXT CAM**.
2. Разрешите браузеру открыть NZXT CAM.
3. Подтвердите **Load Web Integration**.
4. Откройте новую карточку Web Integration и выберите **Configure**.
5. Создайте макет; изменения синхронизируются с дисплеем Kraken.

<details>
<summary><strong>Ручная настройка в NZXT CAM</strong></summary>

1. Откройте **NZXT CAM**.
2. Перейдите в **Lighting → Kraken → LCD Display**.
3. Выберите **Web Integration**.
4. Откройте настройки **Custom Web Integration**.
5. Введите:

   ```text
   https://nzxt-esc.pages.dev/
   ```

6. Выберите **Apply**, затем **Add as Card**.
7. Откройте новую карточку и выберите **Configure**.

## Возможности

| Возможность | Что она дает |
|---|---|
| **Редактор свободных макетов** | Перетаскивайте, масштабируйте, вращайте, меняйте слои, блокируйте, переименовывайте и точно размещайте каждый элемент. |
| **Данные датчиков NZXT CAM в реальном времени** | Создавайте собственные экраны CPU, GPU, RAM, температуры жидкости, мощности, частоты и скорости вентиляторов. |
| **Расширенная графика** | Объединяйте радиальные, линейные, круговые и исторические графики датчиков в одном макете. |
| **Анимированные фоны** | Используйте цвета, градиенты, локальные изображения, GIF, MP4-видео, прямые URL медиа, YouTube и Pinterest. |
| **Интеграция Now Playing** | Показывайте обложку альбома, сведения о треке и визуализации, реагирующие на звук, из локального Windows-клиента. |
| **Explore и Library** | Импортируйте пресеты сообщества, редактируйте каждую часть, управляйте избранным и собственной локальной коллекцией. |
| **Приоритет локального хранения** | Пресеты используют LocalStorage; локальные медиа — IndexedDB и остаются на вашем устройстве. |
| **Многоязычный редактор** | Используйте интерфейс на 18 поддерживаемых языках. |

### Элементы оверлея

Текущий редактор группирует элементы оверлея в четыре понятные категории:

| Содержимое | Данные | Время | Аудио |
|---|---|---|---|
| Текст | Датчик | Цифровые часы | Обложка альбома |
| Фигура | Радиальная графика | Аналоговые часы | Текст Now Playing |
| Значок | Линейная графика | Дата | Аудиовизуализатор |
| Стикер | Круговая графика |  |  |
| Изображение | График датчика |  |  |

Где возможно, все элементы используют одинаковый визуальный процесс: выберите элемент в предпросмотре или списке слоев, затем настройте положение, размер, поворот, порядок, стиль и параметры конкретного типа.

### Мониторинг оборудования

Создавайте макеты в реальном времени на основе доступных данных мониторинга NZXT CAM, включая:

`температура CPU` · `нагрузка CPU` · `частота CPU` · `мощность CPU` · `скорость вентилятора CPU` · `температура GPU` · `нагрузка GPU` · `частота GPU` · `мощность GPU` · `скорость вентилятора GPU` · `использование RAM` · `температура жидкости`

Системы с несколькими GPU могут автоматически выбирать активный графический процессор или использовать конкретный GPU. Когда API NZXT CAM недоступен, браузерный редактор показывает демонстрационные значения, поэтому макеты по-прежнему можно создавать и просматривать.

### Фоны и медиа

Используйте сплошной цвет или градиент как основу, затем добавляйте медиа из:

- Локальных файлов PNG, JPG, GIF, WebP или MP4
- Прямых URL изображений и видео
- Видео YouTube
- Медиассылок Pinterest

Фоновое медиа можно позиционировать, масштабировать, подгонять и сочетать с любым оверлейным макетом. Локальные файлы хранятся в IndexedDB и не загружаются NZXT-ESC.

### Пресеты, Explore и Library

- Сохраняйте и организуйте до **20 пользовательских пресетов** в локальной Library.
- Используйте до **40 элементов оверлея** в каждом пресете.
- Импортируйте и экспортируйте редактируемые файлы пресетов для резервного копирования или обмена.
- Просматривайте макеты сообщества через **Explore**.
- Добавьте пресет из Explore в Library, настройте его и сделайте своим.
- Поддерживайте синхронизацию редактирования и рендеринга Kraken между двумя представлениями NZXT CAM.

## Now Playing и аудиовизуализатор

Дополнительный Windows-клиент [NowPlaying.WebSocket](https://github.com/mrgogo7/NowPlaying.WebSocket) работает локально и отправляет данные медиасессии и аудиоспектра в NZXT-ESC через локальное соединение WebSocket.

С его помощью можно добавить:

- **Обложку альбома** текущую иллюстрацию с настройками размера, рамки и углов
- **Текст Now Playing** название, исполнителя или альбом с прокруткой длинного текста
- **Аудиовизуализатор** настраиваемые спектры и формы волны в реальном времени

Интеграция не ограничена Spotify. Приложение считывает поддерживаемые медиасессии Windows и системный звук из браузеров, медиаплееров и других приложений.

## Конфиденциальность и локальное хранение

NZXT-ESC построен вокруг локального хранилища браузера:

- Конфигурация пресетов хранится в **LocalStorage**.
- Локальные изображения и видео хранятся в **IndexedDB**.
- Показания датчиков NZXT CAM и пресеты пользователей не отправляются в аналитику.
- Приложение намеренно не собирает персональные данные.

Рабочий сайт использует **Google Tag Manager** и **Google Analytics 4** для анонимной продуктовой аналитики. **CookieYes** управляет согласием там, где это требуется, а необязательные аналитические cookie включаются в соответствии с выбором пользователя. Сборки для разработки не требуют рабочих аналитических сервисов.

## Разработка

### Локальный запуск

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`. Если NZXT CAM недоступен, редактор использует демонстрационные данные оборудования.

```bash
npm run build   # Проверить типы и создать рабочую сборку
npm test        # Запустить проверки i18n и набор тестов Vitest
```

### Архитектура

<details>
<summary><strong>Структура проекта и принципы проектирования</strong></summary>

```text
src/
├─ core/       Доменные контракты пресетов, оверлеев, элементов и фонов
├─ render/     Общий движок преобразования пресета в модель рендеринга
├─ storage/    Состояние LocalStorage, импорт/экспорт и медиа IndexedDB
├─ platform/   Адаптеры NZXT CAM и локального приложения
├─ sync/       Синхронизация редактора и среды выполнения
├─ i18n/       Типизированные локализованные сообщения и средства перевода
└─ ui/
   ├─ config/  Редактор конфигурации с перетаскиванием
   ├─ kraken/  Легковесная среда выполнения дисплея Kraken
   └─ shared/  Повторно используемые компоненты интерфейса
```

Предпросмотр редактора и среда выполнения Kraken используют один и тот же канонический конвейер рендеринга. Общий движок обеспечивает одинаковые макет, стиль и трансформации между проектом пользователя и изображением на физическом дисплее.

Данные пресетов нормализуются перед сохранением, импорт и экспорт версионируются, а обновления редактора синхронизируются через `BroadcastChannel` с резервным вариантом на `localStorage`.

</details>

## FAQ

<details>
<summary><strong>Нужно ли устанавливать NZXT-ESC?</strong></summary>

Основной редактор не требует отдельной установки. Откройте его через NZXT CAM Web Integration. Только дополнительные музыкальные оверлеи требуют локального Windows-клиента NowPlaying.WebSocket.

</details>

<details>
<summary><strong>Работает ли NZXT-ESC без NZXT CAM?</strong></summary>

Редактор можно открыть в обычном браузере и использовать демонстрационные значения датчиков для дизайна. Мониторинг оборудования в реальном времени и вывод на дисплей Kraken требуют NZXT CAM Web Integration.

</details>

<details>
<summary><strong>Какие модели NZXT Kraken поддерживаются?</strong></summary>

NZXT-ESC предназначен для устройств NZXT Kraken, поддерживающих режим дисплея NZXT CAM Web Integration. Доступный размер и форма дисплея определяются через API NZXT CAM.

</details>

<details>
<summary><strong>Где хранятся пресеты и локальные медиа?</strong></summary>

Пресеты хранятся в LocalStorage браузера. Локальные изображения и видео — в IndexedDB. Регулярно экспортируйте важные пресеты при смене браузера, установки Windows или компьютера.

</details>

<details>
<summary><strong>Для Now Playing нужен Spotify?</strong></summary>

Нет. NowPlaying.WebSocket использует поддерживаемые медиасессии Windows и системный звук, поэтому может работать с браузерами и другими совместимыми медиаприложениями.

</details>

<details>
<summary><strong>Можно ли редактировать пресеты сообщества?</strong></summary>

Да. Пресеты, импортированные из Explore, полностью редактируются после добавления в Library.

</details>
